# [PR 700](https://github.com/vig-os/devcontainer/pull/700) build(nix): expose libstdc++ in dev-shell so pymarkdown runs on NixOS

## Description

The `pymarkdown` pre-commit hook failed on a NixOS host with
`ImportError: libstdc++.so.6: cannot open shared object file`, forcing
`--no-verify` on every local commit. `pymarkdown` runs from pre-commit's **own**
manylinux-wheel Python env (not the project venv); its dependency `pyjson5` is a
C extension linked against `libstdc++.so.6`, which a NixOS host does not put on
the loader path outside an FHS environment. Unlike the standalone binaries in
sibling issue #697, `pymarkdown` is **not** packaged in nixpkgs, so the "add to
`devTools` + `language: system`" recipe does not apply.

This PR makes the existing manylinux wheel resolve its C library by exposing the
Nix C++ runtime on the dev-shell loader path.

## Changes Made

- `flake.nix` (`mkProjectShell`): append `${pkgs.stdenv.cc.cc.lib}/lib` to
  `LD_LIBRARY_PATH` via the shell hook, **preserving** the value mkShell already
  injects (it auto-populates `LD_LIBRARY_PATH` from some packages' propagated
  libs, e.g. `alsa-lib`) and any host-set value. This is the **same** libstdc++
  the Nix toolchain itself links, so there is no version clash with the other
  dev-shell binaries, and it generalises to any future C-extension Python hook.
- `tests/test_flake_devshell.py`: two dev-shell parity tests (mirroring the
  existing `dev_shell_env` / `BATS_LIB_PATH` tests) asserting `LD_LIBRARY_PATH`
  carries `libstdc++.so.6` and that the C library loads under the dev-shell
  loader.
- `docs/NIX.md`: documents the mechanism (and notes `nix-ld` as a non-enforceable
  fallback) under the pre-commit decisions section.
- `CHANGELOG.md` (+ scaffolded mirror via `sync_manifest`): `### Fixed` entry.

### Approach chosen and alternatives rejected

- **Chosen — Option 1 (dev-shell `LD_LIBRARY_PATH`).** Smallest, robust change;
  keeps the upstream `rev:` pin; verified that pymarkdown runs and that the other
  Nix binaries (`just`, `nixfmt`, `jq`, `rg`, `git`, `nvim`, `lazygit`, `delta`,
  `bat`, `eza`, `taplo`, `shellcheck`, `claude`) still run with it set. Used
  `LD_LIBRARY_PATH` rather than `NIX_LD`/`NIX_LD_LIBRARY_PATH`: the latter only
  helps binaries launched through the NixOS `stub-ld`, whereas the wheel's C
  extension is `dlopen`'d by an already-running Nix-store CPython that honours
  `LD_LIBRARY_PATH` directly — the more general and host-independent lever.
- **Rejected — Option 2 (package `pymarkdown` in the flake).** Most hermetic, but
  `pymarkdown` is absent from nixpkgs, so it means a `buildPythonApplication` from
  the PyPI sdist plus a version to maintain — more work than warranted to unblock
  the hook, and the env fix is clean and robust. Left as a possible later
  hermetic end-state.
- **Rejected as the fix — Option 3 (`nix-ld` host config).** Per-contributor
  system config the repo cannot enforce; documented as a fallback only.

### Manual Testing Details

All run on the NixOS host inside `nix develop`:

- **Reproduced the bug** (before the fix):
  `nix develop -c bash -c 'uv run pre-commit run pymarkdown --all-files'`
  → `ImportError: libstdc++.so.6: cannot open shared object file`.
- **TDD RED:** the two new tests failed before the flake change
  (`AssertionError: libstdc++.so.6 not found under LD_LIBRARY_PATH:
  /nix/store/...-alsa-lib.../lib`), confirming mkShell already set a value the
  fix had to preserve.
- **Fix verified — hook passes:**
  `nix develop -c bash -c 'uv run pre-commit run pymarkdown --all-files'` →
  `pymarkdown ... Passed` (no manual env export, no `--no-verify`).
- **Parity tests GREEN:** `uv run pytest tests/test_flake_devshell.py` →
  `8 passed`.
- **No regression to other Nix tools:** `just`, `nixfmt`, `jq`, `rg`, `git`,
  `nvim`, `lazygit`, `delta`, `bat`, `eza`, `taplo`, `shellcheck`, `claude` all
  run with `LD_LIBRARY_PATH` set (`ruff` is absent on this branch only because it
  joins `devTools` in #697 — not a regression here).
- **Flake clean:** `nixfmt --check flake.nix` clean; `nix eval` of
  `devShells`/`checks` evaluate; `nix build .#checks.x86_64-linux.format`
  succeeds.

**`--no-verify` rationale:** the `pymarkdown`/`ruff`/`typos` pre-commit hooks
cannot run on this NixOS host pre-fix (the very bug being fixed, plus #697), so
the TDD commits used `git commit --no-verify`. CI re-runs the full hook suite
under `provision-via-flake`. The new markdown was validated manually:
`pre-commit run pymarkdown --files CHANGELOG.md docs/NIX.md` → Passed.
