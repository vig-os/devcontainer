# [PR 797](https://github.com/vig-os/devkit/pull/797) ci: flake checks as CI Tier 0 via nix-fast-build (#779)

## Summary

Lands **Wave C / roadmap thread C (#779)**: make the flake's `checks.<system>` the **Tier-0** CI gate, run by **`nix-fast-build`** (parallel + eval-cached) instead of a serial `nix flake check`, with effectful jobs staying GitHub-Actions-orchestrated.

> **Stacked on #778** (base `feature/778-git-hooks-prek`) because it consumes #778's `checks.pre-commit`. Will retarget to `dev` once #778 merges.

### Changes
- **`packages.<system>.nix-fast-build`** ŌĆö a pinned-nixpkgs passthrough, so CI runs `nix run .#nix-fast-build` reproducibly. Deliberately **not** in `devTools` (no image/dev-shell bloat; it has no `--version` so it would trip the parity check).
- **`ci.yml` `project-checks`** ŌĆö the `nix flake check` build step becomes `nix run .#nix-fast-build -- --skip-cached --no-nom --flake .#checks.x86_64-linux`, building treefmt-formatting, deadnix/statix, devShell, devShellTools, and the git-hooks.nix `pre-commit` gate in parallel. A cheap `test_flake_checks.py` step validates the output schema (formatter/checks/apps/modules) that nix-fast-build doesn't build ŌĆö preserving what `nix flake check` covered.
- **`docs/rfcs/ADR-flake-checks-tier0.md`** ŌĆö records the pureŌåÆTier-0 / effectfulŌåÆGH-orchestrated tiering, why the repo/git-dependent pytest units and impure hooks stay GH jobs, and the driver decision (nix-fast-build now; garnix documented as a future option; Tier-1 self-hosting deferred).
- **`test_flake_checks.py`** ŌĆö guard test that the `nix-fast-build` driver package stays exposed.

### Verification
- `nix run .#nix-fast-build -- --flake .#checks.x86_64-linux` ŌåÆ builds all checks, exit 0.
- `tests/test_flake_checks.py -k "not flake_check_succeeds"` ŌåÆ 5 passed (schema + driver guard).
- Pre-commit (prek) clean.
