# [PR 699](https://github.com/vig-os/devcontainer/pull/699) build(nix): source ruff/ruff-format/typos pre-commit hooks from the flake

## Description

On a NixOS host, the `ruff`, `ruff-format`, and `typos` pre-commit hooks could
not run because they were sourced from upstream `astral-sh/ruff-pre-commit` and
`crate-ci/typos` as generic-linux (manylinux) wheels, which NixOS cannot execute
out of the box (no FHS `ld-linux`). This forced `--no-verify` on every local
commit. This PR sources those tools from the Nix flake (the #625 toolchain
SSoT) via `language: system` hooks, consistent with the other local hooks
(`taplo`, `just`, `nixfmt`, `uv run ...`).

Sub-issue of the #625 Nix-migration epic.

### Manual Testing Details

Run on a NixOS host (`export NIX_CONFIG="experimental-features = nix-command flakes"`):

- `uv run pytest tests/test_flake_devshell.py` → 7 passed (new
  `test_devshell_provides_precommit_binary_hooks` was RED before the flake
  change, GREEN after).
- `nix develop -c ruff --version` → `ruff 0.15.14`;
  `nix develop -c typos --version` → `typos-cli 1.46.3`.
- `nix develop -c bash -c 'uv run pre-commit run typos --all-files'` → Passed
  (resolved from the flake).
- `nixfmt --check flake.nix` → clean; `nix eval` of
  `.#devShells.<sys>.default.drvPath` and `.#checks.<sys>` evaluate
  (`devShell`, `devShellTools`, `format`).

Commits used `git commit --no-verify`: the very hooks being fixed
(`ruff`/`ruff-format`/`typos`) cannot execute on this NixOS host pre-fix, which
is the bug. CI re-runs the full pre-commit suite under `provision-via-flake`
(FHS runner) so the PR is still gated.

**Reviewer note — known gap for `ruff`/`ruff-format`:** `ruff` is also a
project Python dependency (`pyproject.toml` `lint`/`devcontainer` groups), so
`.venv/bin/ruff` (a manylinux wheel) exists. Both `justfile` (`just lint` /
`just format` / `just precommit`) and CI launch pre-commit via `uv run`, which
puts `.venv/bin` first on `PATH` and **shadows** the flake's `ruff`. On a NixOS
host this means the `ruff`/`ruff-format` hooks still hit the broken venv binary
under `uv run pre-commit`, whereas `typos` (not a venv dep) resolves cleanly
from the flake. Fully closing the `ruff` case requires removing `ruff` from the
project venv and repointing `justfile`'s `lint`/`format` recipes at the flake
`ruff` (and refreshing `uv.lock`) — changes that touch `pyproject.toml`,
`uv.lock`, and `justfile`, which are outside this issue's stated scope (and
overlap with sibling #698). Flagging for a decision: extend this PR, or split a
follow-up. CI stays green regardless because the FHS runner can execute the venv
ruff wheel.
