# [PR 919](https://github.com/vig-os/devkit/pull/919) fix(workspace): namespace justfile.gh git helpers to prevent consumer recipe collisions

## Summary

Fixes #915 — the scaffold-managed `justfile.gh` shipped bare `log` / `branch` recipes. A consumer whose preserved `justfile.project` also defined `log`/`branch` hit a hard `just` parse failure after upgrade (`recipe log … is redefined`), making `just` unusable and silently disabling the #877 base-recipe repair. Reproduced on a real consumer (mat) both from a clean 0.3.3 base and from its 0.4.0 branch during 0.4.1-rc1 testing.

## Change

Renamed the shipped git-helper recipes at the SSoT root `justfile.gh`:
- `log` → `gh-log`
- `branch` → `gh-branch`

matching the existing `gh-issues` convention, so scaffold recipes can no longer collide with arbitrary consumer recipe names. Regenerated the template (`assets/workspace/.devcontainer/justfile.gh`) via `just sync-workspace` and the embedded recipe listings in `README.md` / `CONTRIBUTE.md` via `generate-docs`. No live internal callers referenced the old names.

## ⚠️ Migration (user-facing)

Consumers who scripted against the shipped `just log` / `just branch` recipes must switch to `just gh-log` / `just gh-branch`. Consumers who defined their **own** `log`/`branch` recipes now keep them without collision — which is the bug being fixed.

## Verification

- New BATS tests (reproduce the `redefined` collision against old names; assert no bare `log`/`branch` in `justfile.gh`); `bats tests/bats/just.bats` → 39/39.
- `just --list` parses; `just sync-workspace` self-consistent (`sync-manifest` gate green).
- Full `prek run --all-files` green locally.

Refs #877.
