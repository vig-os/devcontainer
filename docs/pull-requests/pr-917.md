# [PR 917](https://github.com/vig-os/devkit/pull/917) fix(skills): correct broken links, dup sections, name/title mismatches, drop commands/

## Summary

Fixes five categories of correctness bugs in `.claude/skills/` (issue #912). No behaviour changes — all edits are documentation/link/structure fixes.

### Fix 1 — Broken relative links to `../../rules/*.mdc`

`.claude/rules/` no longer exists (removed in #626). Every link pointing there is dead. Rewrote to correct targets:

| Old link | Correct target |
|---|---|
| `../../rules/branch-naming.mdc` | `../branch-naming/SKILL.md` |
| `../../rules/subagent-delegation.mdc` | `../subagent-delegation/SKILL.md` |
| `../../rules/tdd.mdc` | `../tdd/SKILL.md` |

Files affected: `ci_check`, `code_execute`, `code_review`, `code_tdd`, `git_commit`, `issue_claim`, `issue_triage`, `pr_create`, `pr_post-merge`, `pr_solve`, `worktree_brainstorm`, `worktree_ci-check`, `worktree_ci-fix`, `worktree_execute`, `worktree_plan`, `worktree_pr`, `worktree_solve-and-pr`, `worktree_verify`.

### Fix 2 — Duplicate `## Delegation` sections

`ci_check`, `worktree_ci-fix`, and `worktree_verify` each had the entire `## Delegation` block copy-pasted twice. Removed the second occurrence in each.

### Fix 3 — Skill name mismatch in `solve-and-pr`

Launcher called `just worktree-start <n> "/worktree-solve-and-pr"` (hyphen). Actual skill dir is `worktree_solve-and-pr` (underscore). Fixed to `/worktree_solve-and-pr`.

### Fix 4 — PR-title contradiction

`pr_create/SKILL.md` says do NOT put the issue number in the title. `worktree_pr/SKILL.md` mandated `<type>: <description> (#<issue_number>)`. Aligned `worktree_pr` to match `pr_create`: title is now `<type>: <description>` without the issue number. **Decision rationale**: `pr_create` gives the reason (GitHub appends `(#PR)` automatically); this is the correct policy.

### Fix 5 — Drop obsolete `commands/` wrappers

Every `.claude/commands/*.md` was a 3-4 line stub pointing to the SKILL.md. Skills provide `/X` directly; stubs are dead weight. Commands with extra context lines (e.g. `code_tdd → tdd/SKILL.md`, `git_commit → CLAUDE.md`) are already covered in the SKILL.md files after Fix 1. Deleted the entire `.claude/commands/` directory and updated `CLAUDE.md` command table from `/project:X` to `/X`.

### Assets sync

`assets/workspace/.claude/skills/` was resynchronised via `sync_manifest.py`. Verified file-level identity before committing.
