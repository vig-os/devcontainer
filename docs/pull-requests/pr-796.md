# [PR 796](https://github.com/vig-os/devkit/pull/796) docs: record org nix dev-environment strategy ADR

## What

Adds `docs/rfcs/ADR-nix-devenv-strategy.md` — a decision record for the org's Nix
dev-environment strategy, prompted by [exo-pet/exo-fleet#76](https://github.com/exo-pet/exo-fleet/issues/76)
(devenv→mkShell port) and its "compare devenv / devshell / mkshell / direnv" thread.

The ADR's core move: that four-way question is a **category error**. It conflates
**three separable axes**:

1. **Activation** — `nix-direnv` (auto on `cd`, GC-rooted) vs manual `nix develop`. Complementary.
2. **Shell definition** — `pkgs.mkShell` vs `numtide/devshell` vs `cachix/devenv`. Exclusive per repo.
3. **Local services** — `devenv up`-style MinIO/Postgres orchestration, which devenv *bundles* into axis 2.

## Decisions recorded

- **Axes 1–2 (ratify existing practice):** `pkgs.mkShell` (via `mkProjectShell`) + `nix-direnv`.
  Reject `devenv`/`numtide-devshell` as the shared builder — both interpose their own
  package-declaration surface between the `devTools` SSoT and `PATH`, complicating the
  dev-shell↔image **parity test**; devenv additionally pays the measured ~165s IFD cold-eval
  (#76) and drags its own substituter. Consistent with #27 (already Superseded) and the
  #778/#791 move to `git-hooks.nix` + `prek`.
- **Axis 3 (the one open decision):** adopt `process-compose` + `juspay/services-flake` as the
  nix-native `devenv up` replacement (daemon-free, versions from the same nixpkgs lock, GC-rooted,
  no IFD). Shared `mkProjectServices` helper + a MinIO+Postgres PoC tracked in #795.

Includes two decision matrices (shell-definition; local-services), an Impact/blast-radius section
(**authoritative for `vig-os`**, a **recommendation** to `exo-pet`/`exoma` siblings), and a
`Reconsider if` section.
