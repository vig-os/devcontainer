# [PR 790](https://github.com/vig-os/devkit/pull/790) feat(flake): treefmt-nix, deadnix/statix, modules, and `nix run .#install` (#777)

## Summary

Lands the **A7 flake-polish bucket** (roadmap thread A, PR #670 review), full scope per the plan:

- **treefmt-nix** — `nix fmt` and the flake format gate now run `treefmt` across nix (`nixfmt-rfc-style`), python (`ruff-format`), and toml (`taplo`) in one pass, wrapping the same formatters the pre-commit hooks already run. `checks.format` → `checks.formatting` (`treefmt --fail-on-change`).
- **deadnix + statix** — new `checks.deadnix` / `checks.statix` (scoped to the authored `flake.nix`; the downstream scaffold + example keep their idiomatic `{ self, … }` signatures), and both join `devTools`. Fixed the two findings this surfaced: unused overlay args (`_final`/`_prev`) and `inherit (pkgs) vulnix`.
- **Modules** — `nixosModules.default` / `homeManagerModules.default` install the shared `devTools` toolchain via `programs.vigos-devtools.enable = true`.
- **Install app** — `apps.install`, so `nix run github:vig-os/devcontainer#install` bootstraps a consumer project straight from the flake (`install.sh` stays the behavior SSoT).
