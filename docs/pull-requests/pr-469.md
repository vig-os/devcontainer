# [PR 469](https://github.com/vig-os/devcontainer/pull/469) fix(ci): decouple sync-main-to-dev sync job from dev setup-env

## Description

Fixes the `sync-main-to-dev` workflow failing on the **sync** job when `main` has a newer `setup-env` (e.g. `install-python` and the `retry` helper) than `dev`: that job checks out `dev`, so it was loading an older composite action and then running `retry` without it defined (exit 127). The **sync** job no longer calls `./.github/actions/setup-env`; it exports the same `retry` shell helper inline (aligned with `setup-env`) and documents the chicken-and-egg constraint. Root and workspace template changelogs include the **#459** entry.

The inline `retry` helper is very close to `.github/actions/setup-env`, but `setup-env` also preserves/merges any existing `BASH_ENV` before writing the new helper. Here `BASH_ENV` is always overwritten, so the comment "inline retry matches setup-env/action.yml" is slightly misleading. Either copy the merge logic from `setup-env` or tweak the comment to clarify the intentional difference.
```suggestion
      # are on the runner; this inlines a minimal retry helper similar to setup-env/action.yml
      # but intentionally overwrites BASH_ENV instead of merging with any existing value.
```
