# [PR 472](https://github.com/vig-os/devcontainer/pull/472) ci: downstream promote-release, GHCR cleanup, and RC prune

## Description

Implements issue #463: downstream `promote-release.yml` workspace template paired with root `promote-release`, optional `create-release` for candidate runs (wired through smoke-test dispatch and `just publish-candidate`), best-effort GHCR and git RC cleanup in root `promote-release.yml` after merge, documentation updates (`RELEASE_CYCLE`, `DOWNSTREAM_RELEASE`, `CROSS_REPO_RELEASE_GATE`), and removal of the dead downstream RC gate from upstream `release.yml` (final verification lives in `promote-release`). Adds BATS/integration coverage for promote-release and `create-release`.

Stale RC and `sha256-*`-only package versions are pruned only via the promote workflow (org Packages API); there is no checked-in manual prune script.
