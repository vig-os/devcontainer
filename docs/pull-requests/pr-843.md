# [PR 843](https://github.com/vig-os/devkit/pull/843) fix(ci): run release build-and-test on ubuntu-24.04 and capture vulnix exit

## Summary

Forward-fix for the two `release.yml` defects that failed the first 0.4.0 candidate run ([rc1 failure issue #842](https://github.com/vig-os/devcontainer/issues/842), diagnosis posted there). No tag was consumed — publish was skipped, so the next candidate dispatch is still `0.4.0-rc1`.

1. **Stale runners**: the build-and-test matrix pinned Debian-era `ubuntu-22.04`/`ubuntu-22.04-arm` (podman 3.4.4), whose rootless volume UID mapping breaks `init-workspace.sh`'s in-container rsync (`chmod: Operation not permitted`, exit 23 → pexpect EOF in the `initialized_workspace` fixture). The identical `test-integration` action is green on `ubuntu-24.04` (podman 4.9.3) in `ci.yml` and `nix-image.yml` on the same commit. Switched to `nix-image.yml`'s exact runner expression.
2. **Uncaptured vulnix exit**: the `Run vulnix` step lacked `security-scan.yml`'s `|| rc=$?` capture; under the runner's default `bash -e`, vulnix's normal exit 2 (findings present — all triaged in `.vulnixignore`) aborted the step before `vulnix-gate` evaluated the register. Adopted the nightly's capture/retry pattern; the original intent is preserved — scan crashes (exit > 2) or invalid JSON still fail the job rather than reading as a clean result.

Changelog entry added to the open `## [0.4.0] - TBD` section.

## Verification

- Root-cause evidence: failing job ran `ubuntu-22.04`/podman 3.4.4 vs green ci.yml job `ubuntu-24.04`/podman 4.9.3 (same commit, same action, same image — reproducible Nix build); vulnix step log shows `Process completed with exit code 2` with the `rc<=2` guard unreached.
- The vulnix pattern is copied from `security-scan.yml:108-122`, which passed today both on schedule-shaped dispatch (run 28709965103) and in the release-PR Security Scan check.
- `just precommit` (full suite, all files): green, exit 0.
- `uv run pytest -k "release or workflow"`: 12 passed; the single failure (`test_uv_venv_workflow`) is a stale local `:dev` image missing `prek` (#778) — image-content test unrelated to this workflow-only diff, passing in CI on the parent commit.
