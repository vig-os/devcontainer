# [PR 523](https://github.com/vig-os/devcontainer/pull/523) fix(ci): repair Renovate preset and validate config in PRs (#520)

## Summary

Resolves the Renovate configuration error that caused issue #520 (Mend stopped opening dependency PRs).

## Changes

- **Preset** ([`assets/workspace/.github/renovate-default.json`](./assets/workspace/.github/renovate-default.json)): split Python `packageRules` so `matchUpdateTypes` and `rangeStrategy` are not in the same rule; rename `baseBranches` to `baseBranchPatterns`.
- **Root and workspace** `renovate.json`: remove `uv` from `enabledManagers`. `uv` is not a registered Renovate manager id; `pep621` covers `pyproject.toml` and `uv.lock` per Renovate docs.
- **CI**: add [`.github/workflows/renovate-validate.yml`](./.github/workflows/renovate-validate.yml) running `renovate-config-validator --strict` when renovate JSON changes.
- **Changelog**: Unreleased entries under Added / Fixed.

## Verification

```bash
npx --yes --package renovate@latest -c "renovate-config-validator --strict renovate.json assets/workspace/.github/renovate-default.json"
```

```diff
+        with:
+          node-version: '20'
+

```

The workflow triggers on changes to any `**/renovate*.json`, but the validator command only checks `renovate.json` and `assets/workspace/.github/renovate-default.json`. This means changes to other Renovate configs (e.g. `assets/workspace/renovate.json` or `assets/smoke-test/renovate.json`) can merge without validation. Please include all Renovate config entrypoints in the validation command (or discover them dynamically) so the path filter and validation coverage match.
