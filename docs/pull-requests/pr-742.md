# [PR 742](https://github.com/vig-os/devkit/pull/742) feat(image): bake /etc/nix/nix.conf enabling nix-command and flakes

## Summary

The Nix-built devcontainer image bundles CppNix but shipped no `/etc/nix/nix.conf`, so the modern CLI's `nix-command`/`flakes` experimental features were disabled by default. Ad-hoc on-demand tooling (`nix shell nixpkgs#<x>`, `nix run`, `nix eval`) failed without an explicit `--extra-experimental-features "nix-command flakes"` flag.

## Change

- `flake.nix`: the `buildLayeredImage` bootstrap layer now writes `/etc/nix/nix.conf` containing:
  ```
  experimental-features = nix-command flakes
  accept-flake-config = true
  ```
  `accept-flake-config` matches the repo's build posture (CI and docs invoke `nix build --accept-flake-config`).
- `tests/test_image.py`: new `TestNixConfiguration` suite asserting the file exists, enables both features, and that `nix eval` works without the flag.
- `CHANGELOG.md` (+ synced workspace mirror): `## Unreleased` → `### Added` entry.

Flake-registry `nixpkgs` pinning to the image's locked input is deferred: it needs the input's store path threaded into the bootstrap and is not a one-liner, so on-demand `nix shell` tracks the channel default for now (noted in a code comment).

## Verification

This environment is offline, so the full image build (which fetches its closure from `cache.nixos.org`) could not run locally. Verified instead:
- `nix eval --raw .#devcontainerImage.drvPath` succeeds — the flake change evaluates to a valid derivation.
- `nixfmt --check flake.nix` clean; full `pre-commit` run passes.
- The heredoc indentation (20 spaces) matches the existing proven `.bashrc` heredoc; a dedent+heredoc simulation produces a clean `nix.conf` with no leading whitespace.
- New tests collect cleanly.

The CI `image` suite will perform the real build + in-container assertion that `cat /etc/nix/nix.conf` and `nix eval --expr "1+1"` succeed without the experimental-features flag.

Closes #739
