# [PR 782](https://github.com/vig-os/devkit/pull/782) fix(image): bake explicit substituters/keys instead of accept-flake-config

## Problem

`accept-flake-config = true` was baked into the image's `/etc/nix/nix.conf`. That made any in-container `nix run github:attacker/flake` silently accept that flake's `substituters`/`trusted-public-keys` ŌĆö the textbook Nix cache-redirection supply-chain attack (PR #670 roadmap thread A).

## Fix

- **Remove** `accept-flake-config = true` from the baked nix.conf.
- **Bake explicit** `substituters` + `trusted-public-keys` so normal builds still hit the caches, while a foreign flake's `nixConfig` now requires a per-invocation `--accept-flake-config`.
- Keep `experimental-features` and the empty `build-users-group =` (#750) intact.

Keys/URLs are taken verbatim from the repo's existing source of truth, `CONTRIBUTE.md:44-45` (the public `vig-os` Cachix cache + upstream `cache.nixos.org`). These are PUBLIC keys, safe to bake.

### Baked `/etc/nix/nix.conf` ŌĆö before / after

Before:
```conf
experimental-features = nix-command flakes
accept-flake-config = true
build-users-group =
```

After (verified by extracting the built image tarball's bootstrap layer):
```conf
experimental-features = nix-command flakes
substituters = https://cache.nixos.org https://vig-os.cachix.org
trusted-public-keys = cache.nixos.org-1:6NCHdD59X431o0gWypbMrAURkbJ16ZPMQFGspcDShjY= vig-os.cachix.org-1:yoOYRi3bvnM6ThxO0joLt7vtzhTfkq3r6jykeUMg7Bk=
build-users-group =
```

### Cachix key source

`vig-os.cachix.org-1:yoOYRi3bvnM6ThxO0joLt7vtzhTfkq3r6jykeUMg7Bk=` ŌĆö `CONTRIBUTE.md:45` (the documented `vig-os` Cachix substituter setup). Not invented.

## Tests

Extended `tests/test_image.py::TestNixConfiguration` (alongside the existing #749 `build-users-group` assertions) with three testinfra checks against the baked nix.conf:
- `test_nix_conf_does_not_accept_flake_config` ŌĆö asserts `accept-flake-config = true` is gone.
- `test_nix_conf_bakes_explicit_substituters` ŌĆö asserts both cache URLs present.
- `test_nix_conf_bakes_trusted_public_keys` ŌĆö asserts both public keys present.
