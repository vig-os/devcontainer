# [PR 350](https://github.com/vig-os/devcontainer/pull/350) fix(ci): harden release workflow retries for network-sensitive steps

## Description

Hardens the release workflow against transient network failures that previously caused candidate publishing to fail during Sigstore/Rekor interactions. It adds retries for SBOM generation, cosign signing, provenance/SBOM attestations, and smoke-test repository dispatch, and migrates SBOM attestation from deprecated `actions/attest-sbom` to `actions/attest`.

## Changes Made

- `.github/workflows/release.yml`
  - Add retry loop (`3` attempts, `15s` backoff) around `cosign sign` in `publish` job
  - Add first-attempt + conditional retry pattern for `Generate SBOM`
  - Add first-attempt + conditional retry pattern for `Attest build provenance`
  - Replace deprecated `actions/attest-sbom` with `actions/attest@v4.1.0` and add retry pattern
  - Add retry loop (`3` attempts, `10s` backoff) for smoke-test `repository_dispatch` API call
