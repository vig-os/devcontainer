# [PR 261](https://github.com/vig-os/devcontainer/pull/261) fix(ci): align smoke dispatch payload key and tag validation

## Description

Align the smoke-test dispatch payload key and RC tag validation regex between the devcontainer release workflow and the smoke-test repository dispatch listener. The release workflow was sending `client_payload[rc_tag]` but the receiver expected `client_payload[tag]`, and the tag validation regex used `rc.N` (with dot) while actual RC tags use `rcN` (no dot). Also consolidates duplicate changelog headings that accumulated across prior PRs.

## Changes Made

- `.github/workflows/release.yml` — rename dispatch payload key from `client_payload[rc_tag]` to `client_payload[tag]` to match the smoke-test receiver
- `assets/smoke-test/.github/workflows/repository-dispatch.yml` — fix tag validation regex from `(-rc\.[0-9]+)?` to `(-rc[0-9]+)?` to match actual RC tag format (`X.Y.Z-rcN`)
- `docs/RELEASE_CYCLE.md` — update documentation reference from `client_payload.rc_tag` to `client_payload.tag`
- `CHANGELOG.md` — consolidate duplicate `### Added` / `### Changed` headings and correct `X.Y.Z-rc.N` notation to `X.Y.Z-rcN`
