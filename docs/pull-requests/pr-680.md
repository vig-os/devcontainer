# [PR 680](https://github.com/vig-os/devcontainer/pull/680) feat(nix): add flake formatter + checks and a nix flake check CI gate

## Summary

Adds Nix flake quality gates so the toolchain SSoT (`flake.nix`) is itself gated.

- **`formatter` output** — `pkgs.nixfmt-rfc-style`, so `nix fmt` formats every `*.nix` file idempotently.
- **`nixfmt` pre-commit hook** — `nixfmt --check` on `*.nix` (fails on unformatted files); `nixfmt` is sourced from the flake dev-shell (`nixfmt-rfc-style` added to `devTools`), matching the taplo/shellcheck system-hook pattern.
- **`checks` output** — lightweight flake checks run by `nix flake check`: a `nixfmt --check` format gate, a dev-shell build, and a `devShellTools` eval.
- **CI gate** — a `nix flake check --accept-flake-config` step in the `project-checks` job (Nix + the flake dev-shell are already provisioned there).

The richer pytest dev-shell/image parity test (`tests/test_flake_devshell.py`) is **not** wrapped as a flake check: nix checks build sandboxed without recursive nix access, so a check that itself runs `nix eval`/`nix develop` cannot work. It stays in CI as a pytest; this reasoning is documented in a `flake.nix` comment.

`nix-image.yml` is intentionally untouched (owned by a sibling PR).

## Verification

- `nix flake check --accept-flake-config` → `all checks passed!`
- `nix fmt` is idempotent (no diff on re-run).
- New tests `tests/test_flake_checks.py` (formatter is nixfmt, `checks` exposes the gates, `nix flake check` succeeds) and the existing parity test all pass (5/5).
- `nixfmt` pre-commit hook passes on all three tracked `.nix` files.

Closes #674
