# [PR 807](https://github.com/vig-os/devkit/pull/807) feat(flake): mkProjectServices — daemonless local dev services (process-compose + services-flake)

## Summary

Implements the local-services axis of [ADR-nix-devenv-strategy](docs/rfcs/ADR-nix-devenv-strategy.md) (#794): a shared `lib.mkProjectServices` builder that turns declared [services-flake](https://github.com/juspay/services-flake) modules into a daemonless [process-compose](https://github.com/F1bonacc1/process-compose) stack, exposed as `nix run .#services` from a plain `mkShell` repo — no Docker/Podman daemon, service versions from the pinned nixpkgs lock, zero extra flake inputs downstream.

- **Helper + PoC**: `lib.mkProjectServices` exported beside `mkProjectShell`; this flake carries the validating PoC (`nix run .#services` boots SeaweedFS S3 + Postgres), asserted end-to-end by the new `tests/test_flake_services.py` (boot, health probes, teardown — no container fixtures anywhere).
- **flake-parts decision (AC)**: resolved without adopting it — `process-compose-flake`'s standalone `evalModules` (services-flake's documented no-flake-parts path); the flake stays on flake-utils. Both new inputs are dependency-free leaf lock entries (+2 total).
- **Timing note (AC)**: ~0.8 s eval, ~3 s closure substitute, ~3 s boot-to-healthy (x86_64-linux, warm store) vs devenv's ~165 s IFD cold-eval. Recorded in `docs/NIX.md`.
- **AC deviation — SeaweedFS, not MinIO**: nixpkgs marks `minio` abandoned upstream with six unfixed CVEs (both channels); the blessed PoC uses the maintained S3-compatible SeaweedFS instead. Rationale on the issue ([comment](https://github.com/vig-os/devcontainer/issues/795#issuecomment-4875758851)) and in `docs/NIX.md`.
- **Scaffold opt-in (AC)**: the flake stub documents a commented `packages.services` block; `justfile.project` ships a commented `services` recipe — both preserved-on-upgrade files, so existing consumers are untouched.
- **Verb namespace**: the managed `justfile.devc` compose verbs are renamed `devc-up`/`devc-down`/`devc-status`/`devc-logs`/`devc-shell`/`devc-restart`/`devc-open` (consumer-facing, flagged under Changed). The broader justfile audit is split out to #806.
- `packages`/`apps` move out of the Linux-only `optionalAttrs` guard (a shallow `//` would have clobbered them); only the dockerTools image set stays guarded — guarded by an eval test, and the darwin eval verified.
