# [PR 256](https://github.com/vig-os/devcontainer/pull/256) fix(setup): restore just default recipe listing

## Description

Running `just` with no arguments no longer shows available recipes because the `lint` recipe was placed before the `default` recipe in the justfile. This PR moves `default` back to the first position and adds a regression test. Also bumps expected `gh` CLI version to 2.88.x.

## Changes Made

- `justfile` — reordered recipes so `default` appears before `lint`, restoring `just` with no arguments listing available recipes
- `tests/bats/just.bats` — new bats test asserting `just` output starts with "Available recipes"
- `tests/test_image.py` — bump expected `gh` version from 2.87 to 2.88
- `CHANGELOG.md` — added Fixed entry for #254 and Changed entry for gh 2.88 bump

## Changelog Entry

### Fixed

- **`just` default recipe hidden by lint recipe** ([#254](https://github.com/vig-os/devcontainer/issues/254))
  - The `default` recipe must appear before any other recipe in the justfile; `lint` was placed first, shadowing the recipe listing
  - Moved `default` recipe above `lint` to restore `just` with no arguments showing available recipes

### Changed

- **Bump GitHub CLI to 2.88.x**
  - Update expected `gh` version in image tests from 2.87 to 2.88
