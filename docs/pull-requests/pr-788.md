# [PR 788](https://github.com/vig-os/devkit/pull/788) docs(rfc): record uv2nix/pyproject-nix evaluation and decision

## Summary

Resolves the #775 spike: an ADR evaluating whether adopting **uv2nix** +
**pyproject-nix** would dissolve the flake's FHS/manylinux "scar tissue", or
merely relocate it. Adds `docs/rfcs/ADR-uv2nix-pyproject-nix.md`. Docs-only, no
code change.

## Decision: **Defer (do not adopt now)**

uv2nix genuinely *eliminates* manylinux-wheel loader breakage — but only for the
wheels **inside the project venv it builds**, which today is a problem this repo
does not have (the baked `pythonEnv` deps are all pure Python). The scar tissue
#775 targets lives on two surfaces uv2nix cannot reach:

- **pre-commit's own hook environments** (#698/#703) — pre-commit provisions its
  own venvs at runtime; `pyjson5` is not in our `uv.lock` (upstream-confirmed out
  of scope).
- **the image's runtime substrate for downstream/consumer wheels** (#735/#736) —
  the devcontainer is a base image; uv2nix only knows its own `uv.lock`, not
  consumers'.

…plus two non-Python items (#728 npm, #740 docker shim). Its one genuine win is
tidying the two hand-rolled `buildPythonPackage` derivations behind `pythonEnv`
(#683 is already fixed cheaply by a 2-line `UV_PYTHON` pin). Net: uv2nix **leaves**
far more of the named scar tissue than it **eliminates**, at the cost of 3 flake
inputs, an override overlay, and a regressed live `uv` dev-loop.

### Eliminate vs relocate vs leave (per item)

| Item | Verdict |
|------|---------|
| #683 dev-shell interpreter pin | Eliminate (for the project venv) — already cheaply fixed |
| #698/#703 pre-commit `libstdc++` `LD_LIBRARY_PATH` | **Leave** (out of scope) |
| #735/#736 image FHS loader + `LD_LIBRARY_PATH` | **Leave** (serves runtime/consumer wheels) |
| hand-rolled `pythonEnv` derivations | Eliminate (the genuine win) |
| native-wheel need *if `pythonEnv` gains a C-ext dep* | Eliminate PT_INTERP/RPATH auto; relocate external-lib case to bounded, fail-fast override |
| #728 npm / #740 docker shim | **Leave** (not Python) |

Reconsider triggers and the least-bad entry point (Option C2, image `pythonEnv`
only) are documented in the ADR. No prototype was built: the conclusion turns on
scope (uv2nix cannot see pre-commit or consumer envs), not on a build succeeding,
so a prototype would not change the verdict.

No changelog entry: docs-only ADR with no user-visible code change.
