# [PR 872](https://github.com/vig-os/devkit/pull/872) fix(ci): move NVD mirror to dedicated vig-os/nvd-mirror Pages host

Follow-up to #871 (already merged). Refs #870.

## Why

#871 landed the mirror as **release assets in this repo**, but that approach is blocked by **GitHub immutable releases**: assets can't be updated/clobbered after creation, and once the `nvd-mirror` tag is used by an immutable release it's poisoned (can't recreate). The in-repo `nvd-mirror-refresh.yml` also can't run — a new workflow is only schedulable/dispatchable from the default branch, and `main` is still the pre-cutover Debian world.

So as merged, `release/0.4.0` points `vulnix --mirror` at a release-asset URL that can't exist → the gate would fail.

## Change

- **Remove** the in-repo `nvd-mirror-refresh.yml`.
- Point `vulnix --mirror` (in `security-scan.yml` and the `release.yml` `vulnix-gate`) at the **dedicated public mirror** [`vig-os/nvd-mirror`](https://github.com/vig-os/nvd-mirror), served on GitHub Pages: `https://vig-os.github.io/nvd-mirror/`.
- Update docs + CHANGELOG.

The dedicated repo owns the one job that talks to NVD (its default branch, so it schedules/dispatches immediately — no cutover dependency). It is **already live and serving** all 7 feeds (`nvdcve-2.0-{2021..2026}.json.gz` + `modified`, 117 MB, HTTP 200 + valid gzip), bootstrapped from a known-good local copy.

## Test plan

- [x] `https://vig-os.github.io/nvd-mirror/nvdcve-2.0-*.json.gz` → 200, valid gzip (all 7).
- [x] Dispatch `security-scan.yml` on this branch → **passed in 97 s** via the mirror ([run 28792515788](https://github.com/vig-os/devcontainer/actions/runs/28792515788)); `NVD_MIRROR=https://vig-os.github.io/nvd-mirror/`, gate green, no `nvd.nist.gov` cold-crawl.
