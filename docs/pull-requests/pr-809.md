# [PR 809](https://github.com/vig-os/devkit/pull/809) chore(scaffold): justfile audit — complete devc-* namespacing, reachable worktree recipes, de-stale'd compose docs

## Summary

Audit of the scaffold justfiles for compose-era leftovers post-#799/#795, per #806. The audit confirmed the repo-root justfiles are clean and the `docker-compose.project.yaml`/`local.yaml` layering is an intentional #799 survivor; four findings were actionable and are fixed here:

- **Namespacing completed**: `check` → `devc-check`, `devcontainer-upgrade` → `devc-upgrade` in the managed `justfile.devc` (recipes, hint strings, `version-check.sh` notification text, and exact-string test assertions), freeing the generic `check` name for project use.
- **Worktree recipes were unreachable in consumers**: the scaffold ships `.devcontainer/justfile.worktree` *and* `.claude` skills that invoke `just worktree-start`, but the root justfile never imported it. Now imported via `import?` (direnv-mode safe; verified a git-less scaffold copy still parses — `just` backtick vars are lazy).
- **Dead workflow documented**: the shipped `.devcontainer/README.md` instructed `cp docker-compose.override.yml.example …` — a file that hasn't shipped for a long time. Rewritten to the real project/local layering.
- **Small cleanups**: dropped a vacuous `tests/test_image.py` assertion on a `docker-compose.local.yml` (`.yml`) spelling nothing ships; aligned the dev-env ADR's illustrative services verb with the shipped `just services`; restored the #778 prek changelog entry header that #807's changelog edit accidentally replaced (its sub-bullets were orphaned under the #795 rename entry).
