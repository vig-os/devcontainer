# [PR 485](https://github.com/vig-os/devcontainer/pull/485) fix(ci): use comma-separated FILE_PATHS for commit-action

## Description

Fix `FILE_PATHS` delimiter mismatch in `prepare-release.yml` and `release.yml` workflows. The `commit-action` splits on commas, but `FILE_PATHS` was passed as space-separated values, causing both commit steps to silently no-op with "No files to commit." This PR switches all `FILE_PATHS` values to comma-separated so `commit-action` correctly commits both `CHANGELOG.md` and `assets/workspace/.devcontainer/CHANGELOG.md`.

The regex here only detects a single literal space after `CHANGELOG.md` (and only if the next path starts with `[a-zA-Z]`). This can miss regressions like multiple spaces, tabs, or a second path starting with a non-letter. Consider matching any whitespace-delimited second token (e.g., `CHANGELOG\.md[[:space:]]+[^[:space:]]`) so the test reliably fails whenever multi-file values are space-separated.
```suggestion
    run bash -lc "! grep -E 'FILE_PATHS:.*CHANGELOG\.md[[:space:]]+[^[:space:]]' .github/workflows/prepare-release.yml"
```

Because the grep command is negated (`! grep ...`), this test will also pass if `.github/workflows/prepare-release.yml` is missing/unreadable (grep exits 2, which `!` turns into success). It would be safer to assert the file exists/readable first so the test can’t silently no-op.
