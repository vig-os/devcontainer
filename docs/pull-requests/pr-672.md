# [PR 672](https://github.com/vig-os/devcontainer/pull/672) feat(scripts): make just init Nix-first; retire requirements.yaml

## Description

Make `just init` Nix-first and retire `scripts/requirements.yaml`, closing a gap
in the #625 migration: the contributor bootstrap was still a multi-OS package
installer driven by a second toolchain SSoT (`requirements.yaml`) that had
drifted from `flake.nix` `devTools`.

## Type of Change

- [x] `feat` -- New feature
- [x] `refactor` -- Code restructuring (no behavior change)

## Changes Made

- **`scripts/init.sh`**: rewritten from a ~680-line OS-detecting installer into a
  Nix-first **prerequisite gate + project bootstrapper**. It requires Nix (and
  direnv, unless `--no-direnv`) plus the dev-shell toolchain, then runs one-time,
  idempotent bootstrap (`uv sync --frozen --all-extras`, git hooks path, commit
  template, `pre-commit install-hooks`) with advisory `podman info` / `gh auth
  status` checks. It installs no tools â€” the toolchain is the flake's `devTools` â€”
  and short-circuits inside the built image (`IN_CONTAINER=true`). The gate path
  uses only shell builtins so it works before any tool is on PATH.
- **Retired `scripts/requirements.yaml`** and its `docs/generate.py` consumers
  (`load_requirements`, `format_requirements_table`, `format_install_commands`)
  and their tests. `flake.nix` `devTools` is now the sole toolchain SSoT.
- **Docs**: `CONTRIBUTE.md.j2` "Requirements" table â†’ "Prerequisites: Nix +
  direnv + a working host container runtime (podman/Docker)". Regenerated
  `CONTRIBUTE.md`/`README.md`; updated the `justfile` `init` recipe comment.
