# [PR 244](https://github.com/vig-os/devcontainer/pull/244) refactor: split workspace justfiles into devc and project layers

## Description

This PR restructures the workspace justfile layout by splitting managed devcontainer recipes from project-level recipes. It renames `justfile.base` to `justfile.devc`, introduces `justfile.project`, updates workspace sync/manifest wiring, and adjusts tests/docs to match the new structure.

## Changes Made

- `justfile.base` removed and replaced by `assets/workspace/.devcontainer/justfile.devc` to keep devcontainer-focused recipes in a dedicated managed file.
- Added `assets/workspace/justfile.project` and updated imports in `assets/workspace/justfile` to move project recipes (`lint`, `format`, `precommit`, `test`, `sync`, `update`, `clean-artifacts`, `log`, `branch`) into the project layer.
- Updated sync/config glue (`scripts/manifest.toml`, `assets/workspace/.gitignore`, `.vscode/settings.json`) so the new justfile structure is synced and recognized correctly.
- Updated documentation and tests (`README.md`, `CONTRIBUTE.md`, `tests/test_integration.py`) to validate and describe the restructured justfile architecture.
- Updated `CHANGELOG.md` Unreleased/Changed with issue `#219` entry describing the justfile split.

## Changelog Entry

### Changed

- **Restructure workspace justfile into devc/project split** ([#219](https://github.com/vig-os/devcontainer/issues/219))
  - Rename `justfile.base` to `justfile.devc` and keep devcontainer lifecycle recipes there
  - Move project-level recipes (`lint`, `format`, `precommit`, `test`, `sync`, `update`, `clean-artifacts`, `log`, `branch`) into `justfile.project`
  - Add tracked `justfile.local` template for personal recipes while keeping it ignored in downstream workspaces, and update workspace imports/manifests to the new structure
