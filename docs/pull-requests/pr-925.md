# [PR 925](https://github.com/vig-os/devkit/pull/925) feat(ci): authenticated GHCR probe and opt-in private-image credentials

Closes #920

Split out from the shipped-CI audit (#854, audit item 8) and rides the devkit rename cycle (#781). The shipped container-based CI assumed `ghcr.io/vig-os/devcontainer` was public and unauthenticated; this adds first-class support for private / rate-limited (authenticated) GHCR pulls without breaking public consumers.

## What changed

**1. Authenticated probe in `resolve-image`** (`.github/actions/resolve-image/action.yml`, SSoT; mirror under `assets/workspace/` regenerated by `sync-manifest`)
- New optional inputs `registry-token` / `registry-username` (default `''`).
- When a token is supplied, `docker login ghcr.io` runs before the manifest probe; the anonymous path is kept for public images.
- The probe no longer swallows stderr. On failure it is captured and classified into an actionable `::error::` that distinguishes an **auth/denied** failure (set `GHCR_PULL_TOKEN` / grant `packages: read`) from a **genuinely missing tag**. The tag-resolution step is untouched.

**2. Opt-in credentials on shipped container jobs**
Every shipped workflow job that declares `container: { image: … }` gets:
```yaml
credentials:
  username: ${{ github.actor }}
  password: ${{ secrets.GHCR_PULL_TOKEN || github.token }}
```
plus `packages: read` (added at workflow level where jobs inherit, or in the job's own `permissions:` block otherwise — no existing permission removed). Every `resolve-image` caller passes `registry-token: ${{ secrets.GHCR_PULL_TOKEN || github.token }}` and `registry-username: ${{ github.actor }}`.

Jobs that received a `credentials:` block (16):
- `ci.yml`: `lint`, `test`
- `renovate-changelog-build.yml`: `build` (root SSoT + mirror)
- `release.yml`: `rollback`
- `release-core.yml`: `validate`, `finalize`, `test`
- `release-publish.yml`: `publish`
- `prepare-release.yml`: `prepare`
- `promote-release.yml`: `validate`, `promote`, `merge`, `cleanup`
- `sync-main-to-dev.yml`: `check`, `sync`
- `sync-issues.yml`: `sync`

**3. Docs** — `assets/workspace/docs/container-ci-quirks.md` re-framed from a "public-image limitation" note into first-class authenticated-pull documentation (the `GHCR_PULL_TOKEN` secret contract, the `github.token` fallback, and the `packages: read` requirement).

## Public-consumer safety

The change is opt-in and non-breaking. With `GHCR_PULL_TOKEN` unset, `${{ secrets.GHCR_PULL_TOKEN || github.token }}` falls back to the automatic `GITHUB_TOKEN` — never an empty password — and an **authenticated pull of a public image with `GITHUB_TOKEN` succeeds**. `packages: read` is what lets that automatic token read the package. Private-image consumers set `GHCR_PULL_TOKEN` (or rely on an automatic token with org package access) and the same expression authenticates the pull.
