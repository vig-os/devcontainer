# [PR 368](https://github.com/vig-os/devcontainer/pull/368) fix(ci): decouple upstream sync workflows from ghcr image dependency

## Description

Decouples upstream sync workflows from GHCR image tag resolution so sync automation no longer depends on pre-published devcontainer image tags (for example, `latest`) that can be missing or delayed.

Also removes the obsolete downstream release contract handshake wiring from workspace templates and keeps compatibility enforced through `workflow_call` schemas.

## Changes Made

- Upstream sync workflows now run directly on GitHub-hosted runners without `resolve-image` container coupling:
  - `.github/workflows/sync-issues.yml`
  - `.github/workflows/sync-main-to-dev.yml`
- Reduced setup bootstrap scope in sync workflows by using `setup-env` only for retry helper needs and disabling unnecessary installs (`install-python: 'false'`, `install-just: 'false'`).
- Deleted obsolete upstream resolver and upstream-only config artifacts:
  - `.github/actions/resolve-image/action.yml`
  - `.vig-os`
  - `assets/smoke-test/.vig-os`
- Removed downstream release contract handshake layer from workspace templates:
  - `assets/workspace/.github/actions/validate-contract/action.yml`
  - `assets/workspace/.github/workflows/release-core.yml`
  - `assets/workspace/.github/workflows/release-extension.yml`
  - `assets/workspace/.github/workflows/release-publish.yml`
  - `assets/workspace/.github/workflows/release.yml`
- Kept workspace sync workflows explicitly decoupled from upstream template behavior:
  - `assets/workspace/.github/workflows/sync-issues.yml`
  - `assets/workspace/.github/workflows/sync-main-to-dev.yml`
- Updated docs/manifests/changelog for the decoupling model:
  - `docs/DOWNSTREAM_RELEASE.md`
  - `scripts/manifest.toml`
  - `CHANGELOG.md`
  - `assets/workspace/.devcontainer/CHANGELOG.md`
