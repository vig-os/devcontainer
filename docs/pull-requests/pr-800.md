# [PR 800](https://github.com/vig-os/devkit/pull/800) chore: drop the sidecar / multi-container capability entirely

## What

Remove the **sidecar / multi-container capability** entirely (issue #799, supersedes #719). Post the Nix migration it no longer earns its keep:

- **Toolchains** (compilers, Geant4, analysis libs) belong in Nix devShells (`mkProjectShell` + the future modular `vigos.devShells.{cpp,geant4,…}`), on PATH — not a container you `podman exec` into. This obsoletes the build-in-a-sidecar workflow.
- **Local software services** (Postgres, Redis, MinIO, …) are handled by `process-compose` + `services-flake` per ADR #794, implemented in #795 (`mkProjectServices`) — daemon-free, same nixpkgs lock, no container runtime.
- The residual case (opaque/vendor images, prod-parity, hardware sims) does not justify a bespoke sidecar exec framework plus its fixtures and the lone `hadolint` dependency.

**Kept:** the Docker-out-of-Docker (DooD) socket and the ability to build containers *inside* the devcontainer (`test_simple_image_build` and the socket mount/settings are untouched).

## Changes (4 commits)

1. **test:** delete the sidecar fixtures (`sidecar.Containerfile`, `test-build.sh`, fixtures `justfile`), the `test-sidecar` compose service, the `sidecar_image`/`devcontainer_with_sidecar` conftest fixtures, and `TestSidecarConnectivity`; strip the orphaned sidecar cleanup from general teardown.
2. **refactor:** drop the `just sidecar`/`sidecars` recipe group, the `test-sidecar` filters in the root justfile + `justfile.podman`, the commented `justfile.project` example, the CI sidecar-cleanup step, and the sidecar framing in the scaffold compose files (DooD socket retained).
3. **build:** remove `hadolint` from the flake `devTools` (unused since the Containerfile lint hook went with the Debian path in #642; the deleted sidecar Containerfile was its last Dockerfile-like artifact) + its `docs/NIX.md` prose and `tests/test_image.py` assertions.
4. **docs:** amend `ADR-nix-devenv-strategy` with the containers-vs-devShell-vs-services boundary; strip sidecar prose from the `TESTING`/`CONTRIBUTE` templates (regenerated); add the **Removed** changelog entry (both copies).

## Migration note (downstream)

Consumers that ran project sidecars via `docker-compose.project.yaml` should move **toolchains → Nix devShells** and **local services → `process-compose`/`services-flake`** (#795). Building containers from inside the devcontainer still works (DooD socket retained).
