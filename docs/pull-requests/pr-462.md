# [PR 462](https://github.com/vig-os/devcontainer/pull/462) ci: split final release into publish and promote phases

## Description

Splits the final release into two phases: **publish** (existing `release.yml` final run) and **promote** (new human-gated workflow after downstream smoke-test). Final `release.yml` no longer updates GHCR `:latest` or publishes the draft GitHub Release from the UI path alone; operators run `just promote-release` after `devcontainer-smoke-test` has published its final release for the same tag. The promote workflow validates prerequisites, updates `:latest`, undrafts the GitHub Release, and merges the release PR to `main`.

The cosign verification is currently too permissive: `--certificate-identity-regexp='https://github.com/vig-os/devcontainer/'` would accept signatures produced by any workflow/ref in this repo. To harden the gate, restrict the identity to the expected signing workflow and ref (e.g., `.../.github/workflows/release.yml@refs/heads/dev` or whatever ref is used for the signing run), so promote only accepts artifacts produced by the release workflow path you intend.
```suggestion
              --certificate-identity-regexp='^https://github.com/vig-os/devcontainer/.github/workflows/release.yml@refs/heads/dev$' \
```
