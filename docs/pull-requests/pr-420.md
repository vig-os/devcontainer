`merge-release-pr` runs `gh pr merge`/`gh pr view` without checking out the repo and without setting `GH_REPO`. In this workflow, other jobs that run `gh` without a checkout set `GH_REPO: ${{ github.repository }}`; without that (or `--repo`), the GitHub CLI may not be able to resolve the target repository, causing the merge/poll steps to fail.
```suggestion
    needs: [ready-release-pr, trigger-release]
    env:
      GH_REPO: ${{ github.repository }}
```
