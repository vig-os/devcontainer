# [PR 584](https://github.com/vig-os/devcontainer/pull/584) fix(ci): digest-aware GHCR RC cleanup with loud verification

## Description

Fixes recurring silent failure of the `promote-release.yml` GHCR RC cleanup job. RC image tags (`X.Y.Z-rc*`) were never deleted because `RELEASE_APP` lacked package-admin permission (GitHub returns misleading HTTP 404). The job also used a blanket `sha256-*` rule that would delete live release cosign signatures once permissions were fixed.

This PR switches GHCR deletes to `GITHUB_TOKEN` (with repo Admin on the package), adds digest-aware RC/signature selection via a bats-tested script, and makes the cleanup step fail loudly when RC tags remain.

## Changes Made

- **`.github/scripts/select-ghcr-prune-targets.sh`** — jq-driven selection of RC image versions and matching RC cosign signatures; preserves published release signatures
- **`.github/workflows/promote-release.yml`** — GHCR prune step uses `GITHUB_TOKEN`, calls selection script, idempotent delete with post-pass verification (fails step on leftovers)
- **`tests/bats/select-ghcr-prune-targets.bats`** + **`tests/bats/fixtures/ghcr-versions.json`** — bats coverage for selection logic
- **`docs/RELEASE_CYCLE.md`**, **`docs/DOWNSTREAM_RELEASE.md`** — document `GITHUB_TOKEN` + package Admin grant requirement
