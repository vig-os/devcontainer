# [PR 933](https://github.com/vig-os/devkit/pull/933) feat(workspace): language-neutral scaffold + opt-in Python flake template

## Summary

Makes the copied project scaffold **language-neutral** and ships Python as an
**opt-in** flake template. Historically every consumer inherited a Python
*package* starter (`pyproject.toml`, `src/template_project/`, `tests/`) plus a
uv/ruff/pytest CI contract — wrong for the many non-Python repos (Rust, TS
actions, data mirrors), whose CI even failed on adoption (`just test` →
`uv run pytest` exits 5 with no project).

Closes #929
Closes #930

## What changed

**Language-neutral scaffold (#929)**
- Deleted the Python package starter from `assets/workspace/` (`pyproject.toml`,
  `src/template_project/`, `tests/`).
- Guarded `lint`/`format`/`test`/`test-cov` in `justfile.project` on
  `pyproject.toml` (mirroring the existing `sync` guard), so `just sync|precommit|test`
  and both CI contracts stay green on a repo with no Python.
- Removed the now-dead `src/template_project` rename + test-import rewrite from
  `init-workspace.sh`; `pyproject.toml` stays in `PRESERVE_FILES`, so a consumer
  that brings its own is never clobbered.
- Updated bats + image tests, `MIGRATION.md`, the README template, `manifest.toml`
  (documents that Python files are intentionally never synced), and `CHANGELOG.md`.

**Opt-in Python starter (#930)**
- New `templates/python/` (concrete `example_pkg`, minimal pytest dev group),
  registered as `templates.python`. Restore a Python layout on demand with
  `nix flake init -t github:vig-os/devcontainer#python`.

## Why one PR

#929 and #930 must ship together (removal + restore path), so they are bundled
here; each commit carries its own `Refs:`.

## Verification (local, all green)

- **bats: 112/112**.
- **Non-Python CI proof:** every guarded recipe + `just sync && just test` exit 0
  with no `pyproject.toml`.
- **Template smoke:** `nix flake init -t .#python` → rename `example_pkg` →
  `uv sync` + `pytest` → 1 passed.
- **`nix flake check --no-build`:** all checks pass (validates `templates.python`).
- **Full `prek run --all-files`: green.**
- Image-dependent pytest (`test_install_script.py`, `test_image.py`,
  `test_integration.py`) edited + syntax/lint-clean; exercised by the
  `test-integration` CI job.
