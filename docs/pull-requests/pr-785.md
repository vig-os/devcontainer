# [PR 785](https://github.com/vig-os/devkit/pull/785) test(image): assert nix.conf tokens by exact membership (clear CodeQL url-substring)

Closes #784.

CodeQL flagged two high-severity `py/incomplete-url-substring-sanitization` alerts at `tests/test_image.py:1118,1121` — the #773 `nix.conf` assertions used `"https://cache.nixos.org" in sub_line` (URL-substring-in-string), which CodeQL treats as bypassable host validation.

**Fix:** split the `substituters` / `trusted-public-keys` lines into whitespace tokens (`split("=", 1)[1].split()` — maxsplit=1 preserves the base64 keys' trailing `=`) and assert **exact membership** in the token list. Clears the alerts and tightens the assertion (URL/key must be a whole token, not an accidental substring).

No behaviour change; test-only. Verified: `ast.parse` OK, `ruff check` + `ruff format --check` clean. In-container assertions validated by the dispatched image-suite CI.
