# [PR 926](https://github.com/vig-os/devkit/pull/926) fix(image): bake authoritative built-tag record for imageless upgrades

Closes #921

## Problem
On a bare `podman run … /root/assets/init-workspace.sh` upgrade (no `install.sh`), the scaffolded `.vig-os` stayed pinned to the baked template version instead of the image's real tag — the image carried no authoritative record of its own built tag, and `init-workspace.sh` only stamped the pin when `VIG_OS_VERSION` was set (which `install.sh` forwards but a raw `docker/podman run` does not). Split out of #916 (F5); could not be fixed runtime-only.

## Fix
- **Injection point** — the real publish tag (RCs included) is only known at the workflow layer. `.github/actions/build-image/action.yml` now runs `nix build --impure` with `VIG_OS_VERSION` set to the true `inputs.version`.
- **Flake** — reads `builtins.getEnv "VIG_OS_VERSION"` and bakes it into `/root/assets/VERSION` (and the template `.vig-os`). Under a plain, **pure** `nix build`, `getEnv` returns `""` and the build falls back to the checked-in repo pin — so ordinary/CI builds stay bit-reproducible. Verified: pure vs impure-with-tag produce **different** derivation hashes (override flows through), and pure eval is unchanged.
- **init-workspace.sh** — when no `VIG_OS_VERSION` is forwarded, reads the baked `VERSION_FILE` (default `/root/assets/VERSION`) and stamps `.vig-os` with the real tag. Explicit `VIG_OS_VERSION` still wins; an absent record leaves the pin untouched (unchanged behavior for older images).

## Reproducibility reasoning
`builtins.getEnv` is the ONLY env the build reads, it is empty under pure eval, and the repo-pin fallback covers that path — so `nix build` remains reproducible; only the release action opts into `--impure` to record the true tag.

## Tests
Three behavioral bats tests in `tests/bats/init-workspace.bats` (all pass): record present → pin stamped from it; record absent → pin untouched; explicit `VIG_OS_VERSION` wins over the record. Flake eval validated pure (exit 0) and impure (override changes the derivation hash).
