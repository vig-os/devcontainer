# [PR 764](https://github.com/vig-os/devkit/pull/764) fix: skip-pull image check under docker and run safety via uv

## Summary

Fixes two script bugs flagged in the PR #670 review (M4).

**Bug 1 â€” `install.sh` `--skip-pull` under docker.** The local-image check used the podman-only `$RUNTIME image exists`, which docker lacks; with the error hidden by `2>/dev/null`, `--skip-pull` always failed under docker.

```diff
-    if ! $RUNTIME image exists "$IMAGE" 2>/dev/null; then
+    if ! $RUNTIME image inspect "$IMAGE" >/dev/null 2>&1; then
```

`image inspect` works on both docker and podman.

**Bug 2 â€” `ci.yml` bare `safety`.** The `safety` dependency scan was invoked bare instead of via `uv run`, so it ran outside the uv env where it is installed (unlike the adjacent `uv run bandit`).

```diff
-          safety check --json > safety-report.json || true
+          uv run safety check --json > safety-report.json || true
```

`|| true` is kept to match the bandit idiom â€” this is a non-blocking security scan whose report is uploaded as an artifact; failing the job on scanner noise is out of scope (and switching `check` â†’ `scan` would change auth/behavior, also out of scope; #756 owns broader ci.yml job consistency).

## Tests
