# [PR 808](https://github.com/vig-os/devkit/pull/808) chore(release): 0.4.0 cutover readiness — security-scan dispatch, rollback docs, changelog framing

## Summary

Pre-release readiness batch for the 0.4.0 Nix publish-cutover (#639). Grew from docs/trigger work into the full vulnix-gate hardening after the first-ever dispatched scan exposed real defects.

**Cutover readiness (original scope):**
- **`workflow_dispatch` on `security-scan.yml`** — scheduled runs execute from the default branch (`main`, still the Debian-era scan), so the vulnix nightly had never run; manual dispatch provides the #639 go/no-go evidence.
- **Consumer comms in `docs/MIGRATION.md`** — rollback story (final Debian release is 0.3.9; pin `DEVCONTAINER_VERSION=0.3.9` in `.vig-os`) and the `devcontainer` → `devkit` rename heads-up (#781).
- **CHANGELOG cutover framing** — `## Unreleased` leads with the cutover as a BREAKING consumer change; Security entry documents the scan hardening + CVE triage.
- **Stale `Containerfile` comment** in the scaffold compose fixed (#642 remnant).

**vulnix gate hardening (found by dispatching the scan; NVD was mid-incident, nix-community/vulnix#171):**
- Scan step made errexit-safe: vulnix's documented exit 2 ("advisories found") killed the step under the runner's default `bash -e {0}` before findings ever reached the gate.
- Crash-vs-findings disambiguation: a vulnix NVD-timeout crash exits 1 (same as "whitelisted findings"); the step now accepts a run only when the findings JSON validates, and retries throttled fetches 3× with backoff (ETag-resuming).
- Weekly-keyed `actions/cache` of the NVD database: warm runs fetch only the ~3 MB `modified` feed (this cut the scan from ~25 min of fetch roulette to ~4 min).
- `packages.vulnix` patches upstream's hardcoded 10 s NVD fetch timeout to 60 s.
- Findings artifact uploads `if: always()` so a red gate keeps its triage evidence.

**CVE triage (2026-07 batch):**
- 8 new HIGH/CRITICAL findings (libssh2 ×2, socat, libxml2, gzip, fzf ×2, jq) triaged online against NVD/CERT.PL/GHSA + upstream + nixpkgs branch state; accepted into `.vulnixignore` with per-entry rationale and staggered 30–60-day expiries. None fixable by rev advance today; none realistically exploitable in a single-user interactive dev container.

Also merges `origin/dev` (post-#809).

## Verification

- **Green end-to-end scan run** (dispatch on this branch): https://github.com/vig-os/devcontainer/actions/runs/28698572494 — vulnix gate passes with 35 validated exceptions, SBOM + Trivy awareness scan green, 8.5 min total.
- `uv run vulnix-gate <findings> --register .vulnixignore` green locally against the CI findings artifact; `check-expirations` validates the register.
- Patched vulnix verified locally: built store path carries `timeout=60` in `nvd.py`.
- `prek run --all-files` green.

Refs #639, #642, #781.
