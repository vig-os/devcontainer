# [PR 341](https://github.com/vig-os/devcontainer/pull/341) ci: scope dispatch permissions and align CI docs

## Description

This PR addresses two Copilot findings for the smoke-test `repository_dispatch` workflow template by hardening token permissions and fixing CI trigger documentation drift. It also adds the corresponding `## Unreleased` changelog entry for issue traceability.

## Changes Made

- `assets/smoke-test/.github/workflows/repository-dispatch.yml`
  - Change header comment to reflect both CI workflows triggering on deploy PRs (`ci.yml` and `ci-container.yml`)
  - Reduce workflow-level `GITHUB_TOKEN` scope to `contents: read`
  - Grant `contents: write` only at `publish-release` job level
- `CHANGELOG.md`
  - Add an `Unreleased` `Security` entry for issue `#340`
- `assets/workspace/.devcontainer/CHANGELOG.md`
  - Sync generated workspace changelog via manifest hook

## Changelog Entry

### Security

- **Smoke-test dispatch workflow permissions now follow least privilege** ([#340](https://github.com/vig-os/devcontainer/issues/340))
  - Reduce `assets/smoke-test/.github/workflows/repository-dispatch.yml` workflow token permissions from write to read by default
  - Grant `contents: write` only to `publish-release`, the single job that creates or edits GitHub Releases
