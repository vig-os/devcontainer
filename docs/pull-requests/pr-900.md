# [PR 900](https://github.com/vig-os/devkit/pull/900) docs(release): document that the empty Unreleased stays on release branches

## Description

`docs/RELEASE_CYCLE.md` still described `prepare-release` as stripping the empty `## Unreleased` section from the release branch. Since #590 the workflow deliberately keeps it on both branches — the freeze commit is the stable main↔dev merge base, and the sync merge keeps `## Unreleased` instead of silently dropping it (behavior documented in the `prepare-release.yml` header). Noticed while cutting `release/0.4.1` (PR #898), which correctly carries the empty section.
