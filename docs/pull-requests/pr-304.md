<h2>v0.1.5</h2>
<h3>Fixed</h3>
<ul>
<li>Excluded <code>.git</code> metadata paths during <code>FILE_PATHS</code> directory expansion to prevent malformed Git tree paths (issue <a href="https://redirect.github.com/vig-os/commit-action/issues/15">#15</a>).</li>

<h2>v0.1.4</h2>
<h3>Added</h3>
<ul>
<li>Added <code>ALLOW_EMPTY</code> environment variable support to allow creating signed empty commits when no file changes are detected</li>
<li>Added unit test coverage for empty commit behavior in <code>commitViaAPI()</code> and <code>commit-runner</code> flow handling</li>
</ul>
<h3>Changed</h3>
<ul>
<li>Updated npm dependency overrides to force patched <code>minimatch</code> versions across transitive dependency trees.</li>
<li>Updated <code>commitViaAPI()</code> to support empty commits by reusing the parent tree SHA when <code>ALLOW_EMPTY=true</code></li>
<li>Updated runner behavior to preserve default no-op behavior when no files are detected, unless <code>ALLOW_EMPTY=true</code></li>
<li>Updated README usage examples and environment variable documentation for <code>ALLOW_EMPTY</code></li>
<li>Replaced <code>process.exit(0)</code> with early return in runner for improved testability</li>
</ul>
<h3>Fixed</h3>
<ul>
<li>Fixed missing <code>ALLOW_EMPTY</code> in commit-runner environment variable documentation</li>
</ul>
<h3>Security</h3>
<ul>
<li>Fixed <code>minimatch</code> ReDoS vulnerabilities (<code>CVE-2026-27903</code> / <code>GHSA-7r86-cg39-jmmj</code>) by pinning safe transitive versions via npm overrides.</li>
</ul>
