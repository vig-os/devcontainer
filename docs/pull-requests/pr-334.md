```diff
           curl -sSf "https://raw.githubusercontent.com/vig-os/devcontainer/${TAG}/install.sh" \
             | bash -s -- --version "${TAG}" --smoke-test --force --docker .
 
+          if [ ! -f ".devcontainer/CHANGELOG.md" ]; then
+            echo "ERROR: expected .devcontainer/CHANGELOG.md after install"
```

The installer is executed via a `curl | bash` pipeline without `set -o pipefail`, so a network/HTTP failure in `curl` may not fail the step (the pipeline exit code can come from `bash`). Because this repo likely already contains a `.devcontainer/CHANGELOG.md` from a previous deploy, the subsequent file existence check could pass and you may silently deploy stale content. Add `set -euo pipefail` (or otherwise ensure curl failures abort) before running the pipeline.
