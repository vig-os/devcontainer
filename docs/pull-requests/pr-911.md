# [PR 911](https://github.com/vig-os/devkit/pull/911) feat(workspace): .vig-os declarative manifest with persisted mode/identity and bare mode

## Description

Promotes `.vig-os` from a single-purpose version pin into the project's declarative manifest and adds the new `bare` delivery mode.

- **Manifest**: `DEVKIT_MODE`, `DEVKIT_PROJECT`, `DEVKIT_ORG`, `DEVKIT_REPO` (plus reserved `DEVKIT_MODULES`, #884) are persisted alongside `DEVCONTAINER_VERSION` — flat `KEY=VALUE` format unchanged, `DEVCONTAINER_VERSION` not renamed (#781 owns that).
- **Precedence**: explicit flag/env > `.vig-os` > prompt/default, in both `init-workspace.sh` and `install.sh`; resolved values are written back on every (re)scaffold, so a manifest-bearing repo upgrades with `--force` and **no mode/identity flags** while keeping its shape and names.
- **Legacy inference**: version-only manifests get their mode inferred conservatively from the tree shape on upgrade (wider mode on ambiguity — the #859 combination resolves to `both` — inference printed, interactive runs confirm, persisted afterwards; never reshapes on inference alone).
- **No implicit mode switches**: an explicit `--mode` contradicting the persisted `DEVKIT_MODE` refuses in both scripts, pointing at `--preview` (#886) and the deliberate DEVKIT_MODE-edit-on-a-clean-branch flow. `--preview` and `--smoke-test` are exempt.
- **`bare` mode**: standards-only scaffold (justfiles, `.pre-commit-config.yaml`, `.github/` CI, `pyproject.toml`, `.vig-os`); prunes `.devcontainer/`, `flake.nix`, `.envrc` behind the #738/#859 pre-existence guards; ships a host-native `ci.yml` (`assets/workspace-bare/` overlay): no resolve-image step, no container jobs — pinned `astral-sh/setup-uv` plus `uv tool install rust-just prek` drive the same `just sync|precommit|test` contract. The root justfile's `import?` semantics keep the graph loading without `.devcontainer/` (proven by bats).
- **Preview UX**: a populated pre-existing `.devcontainer/` now appears as an explicit PRESERVED line in direnv/bare reports (previously silently absent).

Closes #885

## Review follow-up

Addresses the review on the torn-manifest window plus the nits, belt and braces:

- **Torn-state fix (defect):** the template `.vig-os` now ships `DEVKIT_MODE=` (empty = unset to every parser), and `init-workspace.sh` persists the resolved `DEVKIT_MODE`/`DEVKIT_PROJECT`/`DEVKIT_ORG` immediately after the template rsync — before any abort-capable step (notably `resolve_github_repository`). An upgrade aborted inside the window (e.g. `--no-prompts` on a legacy tree with a non-github origin) can no longer persist `DEVKIT_MODE=both` in a direnv-shaped repo, so the next `--force` run infers/refuses as if the torn run never happened. `DEVKIT_REPO` stays in the late write-back (only known after origin resolution); `DEVKIT_PROJECT`/`ORG`/`REPO` already shipped empty in the template, so empty-as-unset covers them. Reproduced RED with a new end-to-end bats test (abort, assert manifest not poisoned, re-run, assert `.devcontainer/` is NOT re-added), then GREEN.
- **Pinned bare CI tools (nit):** the bare `ci.yml` now installs `rust-just==1.51.0` and `prek==0.3.11` — the versions the flake toolchain ships — with a comment noting the Renovate/manual bump path.
- **MIGRATION.md (nit):** explicitly flags that `SHORT_NAME`/`ORG_NAME` env vars now suppress the interactive prompts (precedence flag/env > manifest > prompt).
