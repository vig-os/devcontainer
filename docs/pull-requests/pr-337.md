# [PR 337](https://github.com/vig-os/devcontainer/pull/337) ci: harden release dispatch and publish ordering

## Description

This PR hardens release orchestration to reduce flaky cross-repo timing failures, and fixes the workspace changelog sync regression.

It combines:
- CI/release flow hardening for downstream smoke-test interactions (#336)
- Manifest/changelog sync regression fix and test hardening (#333)

## Changes Made

- `.github/workflows/release.yml`
  - Keeps smoke dispatch asynchronous (no downstream completion wait in upstream workflow)
  - Scopes destructive rollback to upstream release stages only
  - Hardens downstream RC prerelease lookup with bounded retries
  - Retries transient API/network failures (404 propagation, 429, 5xx, timeout/network/TLS patterns)
  - Fails fast on non-retryable auth/permission failures (401/403/bad credentials/missing scopes)

- `assets/smoke-test/.github/workflows/repository-dispatch.yml`
  - Adds concurrency controls to avoid overlapping runs for the same release tag

- `scripts/manifest.toml`
  - Removes duplicate root `CHANGELOG.md` sync entry that caused overwrite behavior

- `tests/test_integration.py`
  - Hardens changelog ownership regression coverage:
    - root `CHANGELOG.md` must keep workspace `## Unreleased`
    - root must not include versioned `## [` release headings
    - `.devcontainer/CHANGELOG.md` must include generic semver release headings

- `assets/workspace/CHANGELOG.md`
  - Synced after manifest correction

- `CHANGELOG.md`
- `assets/workspace/.devcontainer/CHANGELOG.md`
  - Updates under `Unreleased` for user-visible behavior changes
