```diff
           app-id: ${{ secrets.RELEASE_APP_ID }}
           private-key: ${{ secrets.RELEASE_APP_PRIVATE_KEY }}
 
+      - name: Checkout repository
+        uses: actions/checkout@de0fac2e4500dabe0009e67214ff5f5447ce83dd  # v6.0.2
+
```

`actions/checkout` requires `contents: read` (or a token with repo read access). In this job, `permissions:` currently sets only `issues: write`, which overrides the workflow-level `contents: read` and will cause checkout to fail. Add `contents: read` to the rollback job permissions (or pass an explicit `token:` that has contents read).
