# [PR 1212](https://github.com/vig-os/devkit/pull/1212) feat(workspace): add DEVKIT_WORKFLOW knob (gitflow default / trunk opt-in)

Implements epic #1205 — a per-consumer `DEVKIT_WORKFLOW` knob selecting the branching/release model. **gitflow stays the unchanged default; trunk is opt-in.**

## What it does
- `DEVKIT_WORKFLOW=gitflow|trunk`, **empty ⇒ gitflow** (existing consumers' `.vig-os` stays byte-identical).
- **gitflow (default):** unchanged — `dev` integration branch, cut `release/X.Y.Z` from `dev`, RC train, finalize→`main`+tag, `sync-main-to-dev` back-merge.
- **trunk:** `feature|bugfix|chore` straight to `main`; releases STILL happen but cut `release/X.Y.Z` **from `main`**, RC on it, merge back **into `main`** + tag. Only the `dev` branch and `sync-main-to-dev.yml` disappear; the whole downstream tag/build/publish/promote chain is unchanged (already `release/* → main`).

## How (design)
Realized **entirely at scaffold time** (mirrors `DEVKIT_MODE`) — an anchored `dev→main` render of the scaffolded workflows/skill/pre-commit plus a `sync-main-to-dev.yml` copy-exclude; NO resolve-toolchain runtime wiring, NO workflow twin. Enabled by the verified fact that every `dev` reference in `prepare-release.yml` is a plain branch literal (incl. the #617 read-after-write guard). ADR: `docs/rfcs/ADR-workflow-model.md`.
