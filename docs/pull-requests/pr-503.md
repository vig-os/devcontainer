# [PR 503](https://github.com/vig-os/devcontainer/pull/503) fix: restore release PR body on rollback after finalize failure

## Description

When finalize fails and the release workflow rolls back the release branch, the release PR description could be left out of sync. This change extends the `rollback` job in `release.yml` so that, for final releases, it restores the PR body from the pre-finalization `CHANGELOG.md` (TBD / prepare-release format) using `RELEASE_APP`, and surfaces the outcome in the failure issue and job summary.

`Generate Release App Token` runs unconditionally, even though it’s only needed for the conditional PR body restoration step. If token generation fails (transient action failure, missing secrets in some execution contexts), the rollback job will stop before creating the failure issue/summary. Consider guarding this step with the same `if` as `restore-pr-body` (or moving token generation into that step) and/or setting `continue-on-error: true` so rollback remains best-effort when PR restoration is not applicable.
