# [PR 218](https://github.com/vig-os/devcontainer/pull/218) refactor(vigutils): migrate shared scripts into vig-utils package entrypoints

## Description

Reorganize the `scripts/` directory by migrating all shared utilities into `packages/vig-utils/` as proper Python modules with CLI entry points. After this change, `scripts/` contains only project-specific container build tooling, while all downstream-shared scripts are importable, testable, and callable by command name via vig-utils entry points.

This resolves the root cause of pre-commit hooks failing in deployed workspaces (#161) — callers no longer reference loose script paths that don't exist downstream; they invoke vig-utils entry points installed via `uv pip install`.

## Additional Notes

**Regarding #185 (make container scripts callable via PATH):** This PR effectively resolves #185 through a different mechanism than originally proposed. Instead of adding a scripts directory to PATH, shared scripts are now callable by command name via vig-utils package entry points (`uv pip install -e packages/vig-utils`). The remaining files in `scripts/` are all project-specific build tooling — there are no "deploy-only" scripts left to reorganize.
