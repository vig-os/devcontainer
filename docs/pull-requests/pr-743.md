# [PR 743](https://github.com/vig-os/devkit/pull/743) fix(image): add docker -> podman compatibility shim

## Description

The Nix-built image shipped `podman` but no `docker` binary, while the consumer `.devcontainer/docker-compose.yml` mounts the socket at `/var/run/docker.sock` and sets `DOCKER_HOST`/`CONTAINER_HOST`. Docker-out-of-Docker worked (podman honors `DOCKER_HOST`), but any recipe/script invoking `docker` literally failed with `command not found`.

This bakes a tiny `docker -> podman` wrapper at `/usr/local/bin/docker` (already on the baked `PATH`) in the bootstrap layer, so docker-literal callers get a working binary without pulling in the Docker engine.

## Type of Change

- [x] `fix` -- Bug fix

## Changes Made

- `flake.nix`: bootstrap layer writes `/usr/local/bin/docker` (`#!<bash>` + `exec <podman> "$@"`) and `chmod +x`. The store paths are Nix-interpolated; the quoted heredoc preserves `$@`.
- `tests/test_image.py`: new `TestContainerRuntime` class asserting `podman` runs, `docker` resolves on PATH, and `docker --version` execs podman.
- `CHANGELOG.md` (+ synced `assets/workspace/.devcontainer/CHANGELOG.md` mirror): `### Fixed` entry.
