`TAG` comes from `repository_dispatch` payload and is written directly to `$GITHUB_OUTPUT` and later reused in a git ref (`chore/deploy-${TAG}`) and a raw.githubusercontent.com URL. With only a non-empty check, a malicious or malformed tag (e.g., containing newlines) can cause output-injection and/or invalid refs/URLs. Consider validating `TAG` against an allow-list regex (e.g., semver tags like `vX.Y.Z` / `vX.Y.Z-rc.N`, and forbid whitespace/newlines), and optionally enforce `SOURCE_REPO` / `EVENT_TYPE` to match the expected sender before proceeding.
```suggestion

          # Disallow any whitespace (including newlines) in TAG
          if printf '%s' "${TAG}" | grep -q '[[:space:]]'; then
            echo "ERROR: tag must not contain whitespace or newlines"
            exit 1
          fi

          # Enforce an allow-list regex for tags, e.g. v1.2.3 or v1.2.3-rc.1
          if ! printf '%s' "${TAG}" | grep -Eq '^v[0-9]+\.[0-9]+\.[0-9]+(-rc\.[0-9]+)?$'; then
            echo "ERROR: tag '${TAG}' does not match required format (expected vX.Y.Z or vX.Y.Z-rc.N)"
            exit 1
          fi
```
