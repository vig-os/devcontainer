# [PR 802](https://github.com/vig-os/devkit/pull/802) perf(image): bake the build-time placeholder manifest into the Nix image

## Summary

`assets/init-workspace.sh` reads a build-time manifest at
`$SCRIPT_DIR/.placeholder-manifest.txt` (`/root/assets/.placeholder-manifest.txt`
in the image) to take a fast substitution path, falling back to a slow runtime
`find`+`grep` over the whole scaffold when it is absent (logging `"Manifest not
found, searching at runtime (slower)"`). The Nix image never generated this
manifest, so workspace init always took the slow path. The old Debian
`Containerfile` produced it via `grep -rl` over the workspace assets.

This bakes the manifest at image build time so the fast path fires. Output is
unchanged — the runtime fallback already produced correct results; this is a
startup-time optimization only.

## Changes

- **`flake.nix`** — the `bootstrap` layer now generates
  `/root/assets/.placeholder-manifest.txt` by `grep -rl`-listing every workspace
  asset carrying a `{{SHORT_NAME}}` / `{{ORG_NAME}}` / `{{GITHUB_REPOSITORY}}`
  token, rewritten to the in-image runtime path (`/root/assets/workspace/...`)
  and `LC_ALL=C sort`ed for bit-reproducibility. Token set mirrors the runtime
  fallback in `init-workspace.sh`; matches how the Debian `Containerfile` built
  it. Excludes `.git` / `.venv` / `.pre-commit-cache`.
- **`tests/test_image.py`** — new `TestFileStructure::test_placeholder_manifest_baked`
  asserting the baked image ships the manifest: present, non-empty, every line an
  in-image workspace path, and a known placeholder-bearing file
  (`pyproject.toml`) listed.
- **`CHANGELOG.md`** — `### Changed` entry under `## Unreleased`.

## Verification

- `nix eval .#packages.x86_64-linux.devcontainerImage.drvPath` — flake evaluates
  clean.
- `nix fmt flake.nix` — 0 changes (formatting gate satisfied).
- Reproduced the exact `grep -rl … | sort` output over the real assets locally
  (18 placeholder-bearing files); the only build-specific step is stripping the
  `$out` store prefix via `sed`.
- `pytest --collect-only` confirms the new test collects.

Caveat: I did not run a full `nix build .#devcontainerImage` / the image-content
test here (needs a full image build + a running podman container). The new
`test_placeholder_manifest_baked` gate runs against the built image in CI.
